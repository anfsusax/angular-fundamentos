Input / Output

Resposta do Filho: {{ respostaDoFilho }}

O que aconteceu aqui?

Neste exemplo, o componente pai envia uma mensagem para o componente filho usando a propriedade @Input(). É como quando Deus direciona uma instrução ao Filho: existe uma origem e um destino.

Depois disso, o filho emite um evento de volta usando @Output() com EventEmitter, dizendo: "Recebi sua mensagem e aqui está minha resposta".

Ou seja, você viu dois fluxos acontecendo:

  • Pai → Filho via @Input() (dados descem)
  • Filho → Pai via @Output() (eventos sobem)

Este padrão é um dos fundamentos do Angular. Ele organiza a comunicação entre componentes de forma previsível, clara e estruturada.

A analogia bíblica deixa isso ainda mais claro: assim como Deus comunica verdades, direção e propósito, o Pai (componente pai) passa o estado. Assim como o Filho responde, o componente filho devolve as ações ou dados.
